Project Reflection: E-Commerce Product Management System
In developing the E-Commerce Product Management System, my primary focus was on establishing a robust, scalable architecture using TypeScript and Object-Oriented Programming (OOP) principles. I implemented a Product base class to encapsulate both the data properties (such as price and category) and the behaviors associated with a product. By utilizing TypeScript's static typing, I ensured that method parameters and return types were strictly defined, which drastically reduced potential runtime errors. I also adhered to the Single Responsibility Principle by decoupling the tax and discount calculation logic into standalone utility modules, keeping the core Product class clean and modular.

One of the main challenges I encountered was bridging the gap between the raw, untyped data fetched from the DummyJSON API and my strictly typed TypeScript classes. The raw API response returned standard JavaScript objects that lacked the custom getFinalPrice() and displayDetails() methods required by the project. I overcame this by implementing a data-mapping layer within the API service. By using the .map() method on the parsed JSON response, I passed the raw properties into the new Product(...) constructor, successfully transforming static data into fully functional class instances.

For asynchronous operations, I utilized async/await to handle the network requests predictably. To ensure application stability, I wrapped these operations in try/catch blocks. I engineered a custom AppError class that extends the native Error object, allowing me to attach specific HTTP status codes to network failures. These were then routed through a dedicated error-handling utility for graceful, informative logging. Ultimately, I chose to focus entirely on perfecting these architectural, asynchronous, and OOP foundations within the Node.js environment, opting to forego the optional UI component to ensure the core backend logic was thoroughly tested and structurally sound within the time constraints.
